Section 01
What Is Ethical Hacking?
Simple version: A company pays you to try to break into their systems before the bad guys do. You find the holes, write a report, they fix them. Everyone wins.

Three Types of Hackers

  • White Hat — Ethical hackers. Work with permission. Get paid legally. That's what you're becoming.
  • Grey Hat — Find vulnerabilities without permission but don't steal anything. Still illegal.
  • Black Hat — Criminals. Break in to steal, destroy, or ransom. Go to jail.

What Ethical Hackers Actually Do Day-to-Day

  • Pen testing — attack a company's systems with their permission
  • Bug bounties — find bugs in apps and get paid per bug found
  • Security audits — review code and configs for weaknesses
  • Red team ops — pretend to be a real attacker, full simulation
  • Social engineering tests — try to trick employees into giving access
Why this pays so well: There aren't enough security people. The average breach costs a company $4.4 million. Paying a pen tester $300/hr to find problems first is cheap by comparison.
Section 02
How Hackers Actually Think
The hacker mindset: Everything was built by humans. Humans make mistakes. Find the mistake.

The Three Questions a Hacker Asks

  • What does this system do? Understand the normal behavior first
  • What happens if I do something unexpected? Systems break at edges
  • What does the developer assume I won't do? Those assumptions are the vulnerabilities

The Attack Surface

Every way into a system is called an "attack surface." The bigger the surface, the more ways in. Think of it like a building:

  • Front door = login page
  • Side door = API endpoints
  • Windows = open network ports
  • Employees = social engineering targets
  • Old door they forgot about = legacy systems nobody maintains

Your job is to find ALL the doors before someone else does.

Why Most Hacks Succeed

  • Unpatched software (old versions with known bugs)
  • Weak passwords (still the #1 way in)
  • Human error (someone clicked a phishing email)
  • Misconfigured systems (set up wrong, left open)
  • Too much trust (internal systems that trust each other too much)
Section 03
The 5 Phases of Hacking
Every hack — ethical or criminal — follows the same basic pattern. Knowing this is half the battle.
01
Reconnaissance (Recon)
Gather information without touching anything. Who owns the domain? What software are they running? Who works there? Social media, DNS records, job listings — all goldmines. This is where most of your time goes.
02
Scanning
Now you touch the target. Port scan to see what services are running. What version are they? Are there known vulnerabilities? This is where nmap lives.
03
Gaining Access
Find a vulnerability and use it to get in. Could be an exploit, a weak password, a SQL injection, or just calling someone and asking nicely (social engineering).
04
Maintaining Access
Real attackers plant a backdoor so they can come back. In ethical hacking you document this is possible and move on. Don't actually do this on real systems.
05
Covering Tracks
Attackers delete logs, hide their presence. In ethical hacking, you DO NOT cover tracks — you document everything and hand it to the client.
Section 04
Your Tools — Explained Simply

🗺️ Nmap — The Map Maker

Scans a network or website to see what's running. Like knocking on every door of a building to see which ones open.

Basic scan
nmap target.com
Find what software + version is running
nmap -sV target.com
Scan all 65,535 ports (thorough)
nmap -p- target.com

🌐 Nikto — The Website Inspector

Scans a website looking for common problems: old software, missing security headers, exposed files, known vulnerabilities.

Basic web scan
nikto -h https://target.com

📂 Gobuster — The Hidden Door Finder

Tries thousands of common page names to find hidden parts of a website. /admin, /backup, /login, /old — stuff developers forgot to remove.

Find hidden pages
gobuster dir -u https://target.com -w /usr/share/wordlists/dirb/common.txt

💉 SQLmap — The Database Prober

Tests if a website's database can be tricked into giving out data it shouldn't. SQL injection is when the database runs commands it wasn't supposed to.

Test a URL for SQL injection
sqlmap -u "https://target.com/page?id=1" --dbs
⚠️ Only use on systems you have permission to test.

🔑 Hydra — The Lock Picker

Tries thousands of passwords against a login page automatically. This is called a "brute force" attack.

Try passwords on SSH login
hydra -l admin -P passwords.txt ssh://target.com

🔍 Whois — The Ownership Lookup

Who owns this domain? When was it registered? What's their contact info? The phone book of the internet.

whois target.com

📡 DNS Tools — The Address Book

DNS is what turns "google.com" into an IP address. You can learn a lot about a company's infrastructure from their DNS records.

Look up all DNS records
dig target.com ANY
Find subdomains via certificate logs
curl "https://crt.sh/?q=%.target.com&output=json"
Section 05
Common Attacks Explained

🎣 Phishing

Send a fake email that looks real. "Your account has been compromised, click here." The link goes to a fake login page that steals credentials. Still the #1 way companies get hacked.

💉 SQL Injection

Websites store data in databases. When a website takes your input and puts it directly into a database query without checking it, you can inject your own commands. Instead of typing your name, you type database commands. The database runs them.

Example: Username field expects "jon" but gets: jon' OR '1'='1 — this tricks the database into logging you in without a password.

🖥️ XSS — Cross-Site Scripting

A website displays content from users without checking it. You inject JavaScript code instead of normal text. When someone else views that page, their browser runs your code. You can steal cookies, redirect users, log keystrokes.

🔐 Password Attacks

  • Brute force — try every possible combination
  • Dictionary attack — try common words and passwords
  • Credential stuffing — use leaked passwords from other sites
  • Rainbow tables — pre-computed password hash lookups

This is why password managers and unique passwords matter. One breach = all your accounts compromised if you reuse passwords.

🕵️ Social Engineering

Hacking the human instead of the computer. Call the help desk pretending to be an employee. Send a USB drive labeled "Salary Info 2026" and leave it in the parking lot. 40% of people plug in found USB drives.

Most major corporate breaches involve social engineering at some point. Technology can be patched. People are harder to patch.

🔄 Man-in-the-Middle (MITM)

Position yourself between two people communicating. You read and potentially modify everything they send each other. Common on public WiFi. This is why HTTPS matters — it encrypts the traffic so even if intercepted it's unreadable.

💣 Buffer Overflow

Programs set aside a chunk of memory for input. If you send more data than expected, it overflows into adjacent memory. Done right, you can overwrite the program's instructions with your own code. Old but still relevant in embedded systems and legacy software.

Section 06
How Defenders Fight Back
Know the defense as well as the offense. CEH tests you on both. And understanding defense makes you a better attacker.

🛡️ Firewalls

Filter traffic based on rules. "Allow port 443 (HTTPS), block everything else." Your first line of defense. Hackers look for misconfigured rules, open ports that shouldn't be open, or ways to tunnel traffic through allowed ports.

🚨 IDS / IPS — Intrusion Detection/Prevention

IDS watches traffic and alerts when something looks wrong. IPS actually blocks suspicious traffic. Think of IDS as a security camera and IPS as a security guard with a taser.

Hackers get around these by moving slowly, blending in with normal traffic, or encrypting their payloads.

🔒 Encryption

Scrambles data so only someone with the key can read it. HTTPS encrypts web traffic. End-to-end encryption means even the service provider can't read your messages. Encryption doesn't stop attacks — it stops attackers from reading what they steal.

🔑 MFA — Multi-Factor Authentication

Something you know (password) + something you have (phone) + something you are (fingerprint). Even if an attacker gets your password, they still can't log in without your phone. Enable this on everything.

📋 Least Privilege

Users and programs only get the access they absolutely need. The janitor doesn't need access to the CEO's emails. If an attacker compromises a low-privilege account, they can't do as much damage.

🩹 Patch Management

Keep software updated. Most successful attacks exploit known vulnerabilities that patches already exist for. Companies that get hacked often had the patch available for months.

Section 07
The Legal Stuff (Read This)
⚠️ This is serious. Unauthorized access to computer systems is a federal crime under the Computer Fraud and Abuse Act (CFAA). People go to prison for this. The line between legal and illegal is permission.

What Makes It Legal

  • Written permission — a signed scope of work or bug bounty terms
  • Defined scope — exactly what systems you're allowed to test
  • Time window — when you're allowed to test
  • Your own systems — test whatever you own
  • Intentional test environments — HackTheBox, TryHackMe, DVWA

What Makes It Illegal

  • Scanning or probing systems you don't own without permission
  • Accessing data you're not supposed to see, even accidentally
  • Going outside the agreed scope during a pen test
  • "I was just curious" — not a legal defense
  • Testing a neighbor's WiFi without asking
The Golden Rule: If you don't have explicit written permission, don't touch it. Even if you mean well. Even if you find something serious. Report it through the right channels (responsible disclosure).

Bug Bounty Programs — Legal + Paid

Companies like Google, Apple, Microsoft, and thousands of others pay people to find bugs. This is 100% legal, done within their published rules. You can make serious money.

  • HackerOne — hackerone.com
  • Bugcrowd — bugcrowd.com
  • Intigriti — intigriti.com
  • Some pay $10,000+ for critical findings
